The main unicode issue I had was with using the fuzzywuzzy package. Fuzzywuzzy was created by the developers at seatgeek.com. It's pretty cool. Fuzzywuzzy can take a string and a list of possible matching strings, and by using a string matching algorithim, return a list of similar strings, or the most similar string.

The mean stays constant throughout but the mode wavers. What is interesting is that the mode approaches but never reaches 8. If a band receives a rating of 8 from Pitchfork, they earn a "BEST NEW MUSIC" badge. Earning this honor helps bands sell more albums, gain more notoriety, and reach new listeners.

A major flaw with my entire approach is evident if you sort the albums by rating. If you notice, most of te following albums were not originally released betweeen 1999 and the present. Next, I wanted to see if Pitchfork's popularity has changed the way it reviews albums. I thought that the ratings that they issued would get lower as the site's popularity increased. But, it seems the opposite happened. As the years progressed the number of poor album reviews decreased.

In [8]:
import matplotlib.patches as mpatches
for year in years:
    temp = p_df[p_df.year == year]
    total = temp.shape[0]
    twos = (temp[temp['rating'] < 3].count())/total
    threes = ((temp[temp['rating'] < 4].count())/total) - twos[0]
    fours = ((temp[temp['rating'] < 5].count())/total) - threes[0]
    bar(year, twos[0], color='r', align='center')
    bar(year, threes[0], color='g', align='center', bottom=twos[0])
    bar(year, fours[0], color='b', align='center', bottom=threes[0])
title('All Pitchfork Album Reviews < 5 Rating')
ylabel('Proportion')
xlabel('Year')
axis([1998.5, 2015.5, 0, .14])
xticks(rotation='vertical')
L2 = mpatches.Patch(color='red', label='< 3')
L3 = mpatches.Patch(color='green', label='< 4')
L4 = mpatches.Patch(color='blue', label='< 5')
legend(handles=[L2,L3,L4])
show()
